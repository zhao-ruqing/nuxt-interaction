# nuxt-interaction 开发文档

## 项目概述

基于 Nuxt 4 的全栈 Web 应用，包含带 GSAP 动效的营销首页、用户注册/登录、以及登录后的控制台页面。

## 技术栈

| 层面 | 技术 |
|------|------|
| 框架 | Nuxt 4 (Vue 3 + Nitro) |
| UI 库 | Element Plus |
| 动画 | GSAP (ScrollTrigger) |
| 数据库 | MySQL 8.0 |
| 认证 | JWT (jose) + bcrypt |
| 样式 | SCSS |

## 数据库设计

### users 表

```sql
CREATE TABLE users (
  id INT AUTO_INCREMENT PRIMARY KEY,
  username VARCHAR(50) NOT NULL UNIQUE,
  password VARCHAR(255) NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## API 接口

### POST /api/auth/register
- 入参：`{ username: string, password: string }`
- 成功：`{ success: true, user: { id, username } }`
- 失败：`{ success: false, message: string }`

### POST /api/auth/login
- 入参：`{ username: string, password: string }`
- 成功：设置 `auth_token` httpOnly cookie，返回 `{ success: true, user: { id, username } }`
- 失败：`{ success: false, message: string }`

### GET /api/auth/me
- 需携带 cookie 中的 auth_token
- 成功：`{ success: true, user: { id, username, created_at } }`
- 失败：`{ success: false, message: string }`

## 认证方案

- 密码使用 bcrypt 哈希存储
- 登录成功后签发 JWT（jose 库），有效期 7 天
- JWT 存入 httpOnly + sameSite strict cookie，防止 XSS
- `app/middleware/auth.ts` 命名中间件保护 `/dashboard` 路由

## 页面设计

### 首页

- Hero 区域：GSAP 文字逐行入场动画，背景视差滚动
- 特性卡片：ScrollTrigger 驱动侧边滑入动画
- CTA 区域：背景固定视差效果
- 导航栏：滚动后毛玻璃效果

### 登录/注册页

- 居中卡片表单，Element Plus 组件
- Element Plus Message 提示成功/失败

### 控制台

- 侧边栏 + 顶栏布局
- 内容区展示欢迎信息 + 用户信息卡片
- 登出按钮（清除 cookie 并跳转首页）

## 注意事项

1. `nuxt.config.ts` 中需显式配置 `serverDir: './server'`，Nuxt 4 默认 `srcDir` 为 `app/`，服务端目录不会被自动识别
2. 锁文件二选一：当前项目使用 npm（`package-lock.json`），需删除 `pnpm-lock.yaml`
3. GSAP + ScrollTrigger 仅在客户端执行，需在 `onMounted` 中初始化
4. SCSS 全局变量通过 `vite.css.preprocessorOptions.scss.additionalData` 注入，使用 `@/` 别名引用

```bash
npm run dev      # 启动开发服务器 (http://localhost:3000)
npm run build    # 生产构建
npm run preview  # 预览生产构建
```
