# 318/行鉴项目开发文档 v3

> 项目启动时间：2026年7月  
> 技术栈：Nuxt 4（全栈一体化）  
> 本文档基于项目启动邮件整理，侧重功能描述

## 一、项目概述

### 1.1 项目背景

行鉴项目是一个集打卡、活动、社交、积分兑换于一体的综合服务平台，通过点位打卡、社交互动和积分激励体系，为用户提供线下探索体验和线上社交互动功能。

### 1.2 项目目标

打造可扩展的多城市运营平台，v1.0 试点版本实现核心功能闭环，支持后续功能迭代与城市拓展。

### 1.3 技术架构说明

全栈基于 Nuxt 4 实现，前端使用 SSR 模式渲染桌面端 Web 页面（保持当前项目的桌面端视觉与交互风格，不做移动端 H5 适配），API 层由 Nitro 引擎统一托管，数据库继续使用 MySQL。整体采用前后端同仓库的全栈一体化架构；排行榜、并发控制与常用配置优先基于 MySQL 事务、索引与统计表实现，后续如并发量增长再评估是否引入专用缓存组件。

## 二、功能模块详细说明

### 2.1 打卡系统（P0）

#### 功能概述

用户在指定点位进行打卡签到，系统通过地理围栏校验确保用户到达现场，支持起点/终点解锁机制，记录个人打卡足迹。

#### 功能清单

| 功能点 | 说明 | 优先级 |
| --- | --- | --- |
| 点位打卡 | 用户在点位附近（围栏范围内）可完成打卡 | P0 |
| 围栏校验 | 基于 GPS 定位判断用户是否在点位围栏半径内 | P0 |
| 起点/终点解锁 | 按路线设计，需先打卡起点才能解锁后续点位 | P1 |
| 个人打卡足迹 | 展示用户所有打卡记录，含时间、点位、照片等 | P1 |
| 多城市扩展 | 架构层面支持不同城市的点位数据隔离与管理 | P0 |

#### 子功能明细

**点位打卡**
- 用户进入打卡页面，查看附近点位列表
- 点击点位后触发定位，系统自动校验用户是否在围栏范围内
- 校验通过后完成打卡，获得积分奖励
- 同一点位每日限打卡一次

**围栏校验**
- 基于 Haversine 公式计算用户坐标与点位中心的距离
- 距离小于围栏半径（可配置，单位米）即判定为有效
- 支持 GPS 定位为主、辅助定位兜底的策略

**起点/终点解锁**
- 每条路线有明确的起点和终点
- 用户首次访问某路线时仅起点可打卡
- 起点打卡成功后沿途点位依次解锁
- 终点打卡完成后整条路线标记为完成

**个人打卡足迹**
- 按时间线展示用户所有打卡记录
- 每条记录包含点位名称、打卡时间、点位照片
- 支持按城市、按类型筛选
- 支持地图模式展示打卡分布

**多城市扩展**
- 点位数据按城市维度隔离
- 用户切换城市后展示对应城市的点位
- 城市级别可配置启用/停用

### 2.2 活动系统（P0）

#### 功能概述

支持线上活动的发布、报名、支付及奖励自动发放。用户付费报名即视为完成，无需人工审核凭证。

#### 功能清单

| 功能点 | 说明 | 优先级 |
| --- | --- | --- |
| 活动报名 | 用户选择活动并填写报名信息 | P0 |
| 微信支付 | 接入微信支付完成报名费用缴纳 | P0 |
| 积分/徽章自动发放 | 报名成功后系统自动发放积分和徽章 | P0 |
| 活动管理后台 | 活动创建、编辑、上下架、数据统计 | P0 |

#### 子功能明细

**活动浏览与报名**
- 活动列表页：展示所有可报名的活动，支持按城市、按时间筛选
- 活动详情页：展示活动介绍、价格、名额剩余、起止时间等信息
- 报名表单：用户填写姓名、联系方式等报名信息后提交
- 报名状态跟踪：待支付、已支付、已退款等状态清晰可见

**微信支付**
- 用户提交报名后发起微信支付
- 支持桌面端 Web 支付流程（如微信扫码支付或其他适合 PC 浏览器的支付方式，具体以支付渠道确认为准）
- 支付结果通过回调通知系统，保证幂等性
- 支付超时未完成的报名记录定时清理，释放名额

**积分/徽章自动发放**
- 支付成功后系统自动增加用户积分
- 积分数值在活动配置中设定
- 活动关联徽章的，同时发放对应徽章
- 发放操作放在事务中，任一失败则整体回滚

**活动管理后台**
- 活动创建：设置标题、描述、价格、名额、起止时间、封面图
- 活动编辑：支持修改活动信息（已有人报名的活动限制修改关键字段）
- 活动上下架：控制活动在前台的可见性和可报名状态
- 数据统计：报名人数、支付金额、转化率等运营指标

### 2.3 好友系统（P0）

#### 功能概述

用户可管理好友关系，组建好友团队，参与团队排名与互动。

#### 功能清单

| 功能点 | 说明 | 优先级 |
| --- | --- | --- |
| 好友管理 | 添加/删除好友，查看好友列表 | P0 |
| 好友团队 | 创建和管理好友团队 | P1 |

#### 子功能明细

**好友管理**
- 搜索添加：通过手机号或用户昵称搜索并发送好友申请
- 扫码添加：通过扫描用户二维码发送好友申请
- 好友确认：收到好友申请后可选择同意或拒绝
- 好友列表：展示所有已确认的好友，支持搜索
- 删除好友：解除好友关系

**好友团队**
- 创建团队：用户可创建一个好友团队并设置团队名称
- 邀请成员：从好友列表中邀请成员加入团队
- 团队管理：队长可移除成员、转让队长、解散团队
- 团队上限：每个团队有成员人数上限（可配置）

### 2.4 社交系统（P0）

#### 功能概述

提供基础社交功能和排行榜激励机制，增强用户粘性和活跃度。

#### 功能清单

| 功能点 | 说明 | 优先级 |
| --- | --- | --- |
| 好友添加（手机号） | 通过手机号搜索添加好友 | P0 |
| 好友添加（扫码） | 通过扫描二维码添加好友 | P0 |
| 好友排行 | 展示好友间的积分/成就排名 | P1 |
| 世界排行 | 全平台用户排名 | P1 |
| 团队排行 | 好友团队之间的排名 | P2 |

#### 子功能明细

**好友添加**
- 手机号搜索：输入对方手机号查找用户并发起好友申请
- 扫码添加：个人中心展示用户专属二维码，对方扫码即可添加
- 隐私控制：用户可设置是否允许通过手机号搜索到自己

**排行榜**
- 好友排行：展示当前用户所有好友的积分排名，突出显示自己的位置
- 世界排行：全平台用户按总积分排名，支持分页加载
- 团队排行：各好友团队按团队总积分排名
- 排行周期：支持总榜和月度榜两种维度
- 更新策略：积分变动时实时写入 MySQL 积分流水，并通过排行榜统计表/定时汇总任务更新榜单

### 2.5 积分商城（P0）

#### 功能概述

用户使用积累的积分兑换实物或虚拟商品，支持完整的商品管理和兑换流程。

#### 功能清单

| 功能点 | 说明 | 优先级 |
| --- | --- | --- |
| 商品管理 | 商品上架、编辑、下架、库存管理 | P0 |
| 积分兑换 | 用户使用积分兑换商品 | P0 |
| 兑换记录 | 查看历史兑换记录及物流状态 | P1 |

#### 子功能明细

**商品展示**
- 商品列表：展示所有可兑换的商品，支持按分类筛选
- 商品详情：展示商品图片、所需积分、库存数量、商品描述
- 商品分类：实物商品和虚拟商品两大类

**积分兑换**
- 兑换流程：用户选择商品→确认兑换→扣除积分→生成订单
- 库存校验：兑换时实时校验库存，防止超兑
- 积分校验：兑换时校验用户积分是否充足
- 防并发：使用 MySQL 事务、行级锁、条件更新与唯一约束防止高并发下的超兑问题

**商品管理后台**
- 商品上架：设置商品名称、类型、所需积分、库存、图片、描述
- 商品编辑：修改商品信息
- 商品上下架：控制商品在前台的可兑换状态
- 库存管理：手动调整库存数量

**兑换记录**
- 用户端：查看自己的兑换历史，包含商品名称、兑换时间、消耗积分、订单状态
- 实物商品：展示物流单号和物流进度
- 虚拟商品：展示兑换码或领取链接

**发货管理**
- 实物商品：运营人员在后台标记发货，录入物流单号
- 虚拟商品：兑换成功后自动发货（发放兑换码或激活权益）
- 订单状态流转：待发货→已发货→已完成/已取消

### 2.6 后台管理系统（P0）

#### 功能概述

为运营人员提供全面的管理后台，覆盖城市、点位、活动、商品、用户、订单、团队等维度的管理能力。

#### 功能清单

| 模块 | 功能 | 优先级 |
| --- | --- | --- |
| 城市管理 | 城市的新增、编辑、启用/停用 | P0 |
| 点位管理 | 点位CRUD、批量导入、地图标注 | P0 |
| 活动管理 | 活动创建、编辑、上下架、数据统计 | P0 |
| 商品管理 | 商品CRUD、库存管理、上下架 | P0 |
| 用户管理 | 用户信息查看、状态管理 | P0 |
| 订单管理 | 订单查询、退款处理、物流管理 | P0 |
| 团队管理 | 团队信息查看、成员管理 | P1 |

#### 子功能明细

**城市管理**
- 城市列表：展示所有已接入城市，支持搜索
- 新增城市：填写城市名称、选择状态
- 编辑城市：修改城市信息
- 启用/停用：停用后该城市相关数据在前台不可见

**点位管理**
- 点位列表：展示所有点位，支持按城市、按类型筛选
- 新增点位：填写点位名称、经纬度、类型、围栏半径、图文介绍
- 编辑点位：修改点位信息
- 批量导入：通过 CSV/Excel 文件批量导入点位数据
- 地图标注：在地图上可视化标注点位位置，支持拖拽调整

**活动管理**
- 活动列表：展示所有活动，支持按城市、按状态筛选
- 创建活动：设置活动信息、价格、名额、起止时间、关联积分和徽章
- 编辑活动：修改活动信息
- 上下架：控制活动前台可见性
- 数据统计：报名人数、收入、转化率等

**商品管理**
- 商品列表：展示所有商品，支持按类型筛选
- 新增商品：设置商品信息、所需积分、库存
- 编辑商品：修改商品信息
- 上下架：控制商品前台可见性
- 库存调整：手动增减库存数量

**用户管理**
- 用户列表：展示所有注册用户，支持搜索
- 用户详情：查看用户基本信息、积分余额、等级、徽章
- 用户状态：启用/禁用用户账号
- 积分操作：管理员可手动调整用户积分（需记录操作日志）

**订单管理**
- 订单列表：展示所有兑换订单，支持按状态筛选
- 订单详情：查看订单完整的流转信息
- 发货操作：实物商品标记发货并录入物流单号
- 退款处理：处理用户的退款申请

**团队管理**
- 团队列表：展示所有好友团队
- 团队详情：查看团队成员列表、团队总积分

**通用后台能力**
- 登录认证：管理员账号密码登录，支持 JWT Token
- 权限管理：按角色分配菜单和数据权限（管理员/运营/只读）
- 操作日志：关键操作记录操作人、操作时间、操作内容
- 数据导出：列表数据支持 Excel 格式导出

## 三、数据需求与准备

### 3.1 点位数据清单

需业务方在上线前提供以下数据：

- 点位名称
- 经纬度坐标
- 图文介绍（文字描述 + 图片素材）
- 类型分类（景点/非遗/美食）
- 围栏半径（米）

需业务方提供：

- 用户等级划分规则
- 积分获取规则（打卡奖励、活动奖励、每日签到等）
- 积分消耗规则
- 徽章获取条件

### 3.3 商品资料

需业务方提供：

- 积分兑换商品清单（实物/虚拟）
- 各项商品所需积分
- 商品库存数量
- 商品图片及描述

### 3.4 UI 设计规范

需业务方提供：

- LOGO
- 品牌色（主色、辅色）
- 字体规范
- 页面设计稿或风格参考

## 四、风险与应对

| 风险项 | 影响 | 应对措施 |
| --- | --- | --- |
| 点位数据量大，导入耗时 | 开发进度延迟 | 提前准备批量导入工具，预留数据清洗时间 |
| 微信支付审核周期不确定 | 支付功能无法按时上线 | 提前提交审核材料，预留备用支付方案 |
| 多城市扩展带来的架构复杂度 | 开发成本增加 | 前期做好数据隔离设计，避免后期重构 |
| 地理围栏精度问题 | 用户体验不佳 | 综合使用 GPS + WiFi 辅助定位，提供手动签到兜底 |
| 积分体系设计不合理 | 经济系统失衡 | 与业务方充分沟通规则，预留参数调整能力 |
